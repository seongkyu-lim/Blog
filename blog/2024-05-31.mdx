- prisma
  - typeORM에서 한단계 더 추상화되어 개발자 친화적인 느낌.
  - 쿼리메소드에 entity를 넣는 방식이 아니며 특정 형식의 객체를 넣는데, 연관관계의 데이터도 객체에 특정형식으로 하나의 로직으로 처리할 수 있다.
    ex)

```
user = {
	email: 'elsa@prisma.io',
	name: 'Elsa Prisma',
	posts: {
		create: {
			title: 'Include this post!',
		},
	},
}

```
