- 배포 전략으로 release브랜치를 활용하고, 모노 레포여서 프로젝트별로 release 브랜치가 존재하여 release 브랜치가 여러개일 떄, 최상위 루트 브랜치로 각 프로젝트 별로 있는 release 브랜치를 활용하는 것이 좋을 까 아니면 main 브랜치를 활요하는 것이 좋을까 ?

  - 프로젝트별로 공통적으로 사용되는 부분들에 대해서 작업이 이루어질 수 있고, 여러 프로젝트들에 대한 작업이 하나의 작업브랜치에서 이루어질 수 있으므로 main브랜치를 유지하여 main브랜치에서 작업브랜치를 따서 작업하고 main에 머지 후, main브랜치를 release브랜치에 머지하는 식으로 하는 것이 좋을 것 같다고 생각했다.

- kotlin의 타입에 붙는 ?는 nullable, non-nullable type을 구분하기 위한 표시이며 typescript에서 메소드의 인자에 붙는 ?는 옵셔널의 의미로 아예 값이 없을 수 있다는 의미이다.
