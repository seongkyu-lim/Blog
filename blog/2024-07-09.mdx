nestjs module - import : 해당 모듈에서 사용하고싶은 외부의 모듈 선언. - controller: 해당 모듈에서 사용되는 컨트롤러 객체. - providers: 해당 모듈에서 사용하고 싶은 클래스 선언. - exports: 해당 모듈을 외부에서 import했을 때 사용가능하게 하고 싶은 클래스 선언.

- 빌더 패턴의 장단점
  - 빌더 패턴의 유연성이 장점이자 단점이라 할 수 있다.
    - 유연하게 생성할 데이터를 세팅할 수 있고 하나의 Builder객체의 메소드들을 체이닝 형식으로 호출함으로써 객체 생성과 선언을 동시에 간단하게 선언할 수 있다. 가독성도 좋다.
    - 이러한 유연성이 단점이 될 수 있는데 반드시 선언해야하는 값을 선언하지 않을 가능성이 있으므로 객체 데이터의 무결성이 꺠질 수 있다. 대안으로 정적팩토리 메소드, 생성자 를 활용할 수 있겠다.
  - 상황에 따라 적합한 것을 사용하는 것이 좋을 듯하다.
