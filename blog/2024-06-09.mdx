1. RealMysql 2권을 읽는 목적과 목표
   - 목적 : 백엔드 개발자로서 단순히 원하는 데이터를 처리하는 것에서 더 나아가 db의 처리과정을 이해함으로서 효율적인 쿼리를 짤 줄 아는 것에 대한 필요성을 느꼈다.
   - 목표 : mySql을 더욱 효율적이고 최적화된 형태로 사용할 수 있게 되기.
2. 쿼리문에 대해 동일한 결과를 얻지만 더 나은 방법이 없을지 고민해보고 쿼리 실행 후 실행 계획 데이터를 조회 해보기.
3. DDL과 DML.
4. MySQL 매뉴얼에 명시된 SQL 문법 표기를 읽는 방법.
5. LIkE 연산자는 와일드카드 문자(%, \_)가 검색어 뒤쪽에 있으면 인덱스 레인지 스캔이 가능하지만 앞쪽에 있으면 불가능하다.
6. BETWEEN과 IN
   - BETWEEN은 크다와 작다 비교를 하나로 묶어 둔 것에 가깝니다.
   - IN은 여러개의 동등 비교(=) 연산자를 묶은 것과 같은 방식으로 처리된다.
   - BETWEEN 연산자는 다른 비교 조건과 결합해 하나의 인덱스를 사용할 때에는 BETWEEN 대신 IN을 사용하는 것이 인덱스를 활용할 수 있기 때문에 유리하다.
